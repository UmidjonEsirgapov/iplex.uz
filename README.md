# iplex.uz

Static website for educational articles in multiple languages (English & Russian).

## Repository Structure

```
/
├── index.html          # Landing page
├── sitemap.xml        # Sitemap for SEO
├── .htaccess          # Apache rewrite rules (clean URLs)
├── en/                # English articles
│   ├── index.html
│   └── *.html (3708 articles)
├── ru/                # Russian articles
│   ├── index.html
│   └── *.html (3708 articles)
├── src/               # Source code (build scripts)
│   ├── build.py              # Main SSG script
│   ├── generate_sitemap.py   # Sitemap generator
│   ├── remove_html_from_urls.py  # URL cleaner
│   ├── colab_generator.py    # Google Colab version
│   └── requirements.txt      # Python dependencies
└── templates/         # Jinja2 HTML templates
    ├── article.html
    ├── index.html
    └── landing.html
```

## Features

- Multi-language support (EN/RU)
- Clean URLs (no .html extension)
- SEO optimized
- Internal linking strategy
- Responsive design with Pico.css

## Deployment

This repository contains the static website files ready for deployment.

### GitHub Pages

1. Push to GitHub
2. Enable GitHub Pages in repository settings
3. Select main branch as source

### Traditional Hosting

1. Upload all files to hosting root directory
2. Ensure `.htaccess` is uploaded (for Apache servers)
3. Submit `sitemap.xml` to Google Search Console

## URL Structure

- Landing: `https://iplex.uz/`
- English: `https://iplex.uz/en/`
- Russian: `https://iplex.uz/ru/`
- Articles: `https://iplex.uz/en/article-slug` (clean URLs)

## Source Code

All build scripts are located in the `src/` directory:
- `build.py` - Main SSG script with parallel processing
- `generate_sitemap.py` - Sitemap generator
- `remove_html_from_urls.py` - URL cleaner utility
- `colab_generator.py` - Google Colab version

### Rebuilding the Site

To rebuild or update the site:

```bash
cd src
pip install -r requirements.txt
python build.py
```

The generated files will be in the `output/` directory (which is then moved to root for deployment).
